# Lessons

One note per implementation session, numbered in study order (`01-scaled-dot-product-attention.md`,
`02-multi-head-attention.md`, ...). These are the written-down version of the
teaching that happens in chat — the repo should be self-sufficient to re-learn from,
especially on mobile.

Each note follows this shape:

```markdown
# NN — Topic

**Phase:** which roadmap phase · **Paper(s):** what this comes from
**Code:** file(s) touched · **Test:** how to see it work

## The problem
What limitation or question motivated this piece. (History first!)

## The idea
The intuition, in plain words, before any math.

## The math
The equations, derived rather than dropped — with shapes.

## The code
Walkthrough of what we wrote and why each decision was made.

## What breaks without it
The negative space: what happens if you remove/naively change this part.

## Open questions
Things to revisit, links forward to future lessons.
```
